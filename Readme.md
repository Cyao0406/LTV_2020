# 研究專案：分析版本與命名規範 (README)

本專案採用 **「維度組合命名法」**，將研究設計拆解為「分組邏輯 (Spec)」與「模型規格 (Model)」，以確保研究流程的可追溯性。

---

## 檔案命名規則 

所有輸出資料夾與檔案建議遵循以下格式：
`S_[分組邏輯]_M_[模型]_[描述標籤]`

### 1. 維度定義

#### **S (Spec) - 分組基準**
定義實驗組 (Treatment) 與對照組 (Control) 的劃分邏輯：
* `S_Lag`: 利用 $t-1$ 期資料區分。
* `S_F2020`: 利用 2020 年作為固定切點區分。

#### **M (Model) - 模型設置**
模型規格由以下元件累加組成（請保持字母順序：T > I > Tr）：
* `T`: **Time FE** (年度固定效應)。*注意：本研究之 Baseline 預設包含 Time FE。*
* `I`: **Individual FE** (個體固定效應)。
* `Tr`: **Trend Adj** (時間趨勢調整項)。

## 常用組合對照表 (Version Dictionary)

| 資料夾/檔案名稱範例 | 分組基準 | 模型包含要素 | 統計意義說明 |
| :--- | :--- | :--- | :--- |
| `S_F2020_M_T` | 2020切點 | Time FE | **Baseline**: 基礎實證模型 |
| `S_F2020_M_TI` | 2020切點 | Time + Indiv FE | 控制不隨時間變化之個體特徵 |
| `S_F2020_M_TI_Tr` | 2020切點 | Time + Indiv FE + Trend | **嚴謹規格**: 同時控制個體特徵與時間趨勢 |
| `S_Prev_M_T` | 前一期區分 | Time FE | 測試不同分組邏輯下的基礎結果 |

---

## 注意事項
1. **關於 Baseline:** 本專案不另設「無固定效應 (No FE)」之版本，所有 `M_T` 即為最簡規格。
2. **排序建議:** 在 M 標籤中，請維持 `T` -> `I` -> `Tr` 的順序，例如 `M_TI_Tr` 而非 `M_Tr_IT`，以利檔案排序與檢索。
3. **新增標籤:** 若需加入新調整項 (如 Clustered SE)，請更新此 README 並定義新代號（例如 `C`）。
